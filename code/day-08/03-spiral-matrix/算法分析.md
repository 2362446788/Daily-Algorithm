## 一、两种遍历实现的逻辑差异

| 对比维度 | **spiralOrder1（左闭右开）** | **spiralOrder（闭区间+条件判断）** |
|---------|----------------------------|-----------------------------------|
| **循环条件** | `while (top < bottom && left < right)` <br> 保证至少 2 行 **且** 2 列，即至少能构成一圈 | `while (top <= bottom && left <= right)` <br> 只要边界没交叉，就继续，能处理所有情况 |
| **四条边的遍历范围** | 每条边只到倒数第二个格子：<br> 上：`[left, right)`<br> 右：`[top, bottom)`<br> 下：`(left, right]` <br> 左：`(top, bottom]` | 每条边遍历整行/整列：<br> 上：`[left, right]`<br> 右：`[top, bottom]`<br> 下：`[right, left]`<br> 左：`[bottom, top]` |
| **角落元素归属** | 严格分配：每个角落只被一条边处理，<br> 上边不处理右上角，右边不处理右下角等 | 每条边都包含所有角落，<br> 靠边界收缩和 `if` 判断来避免重复 |
| **额外条件判断** | 不需要，因为循环条件保证了不会出现 <br> 只剩一行/一列时还去读下边或左边 | 第三步和第四步需要 `if (top <= bottom)` <br> 和 `if (left <= right)`，<br> 防止单行/单列重复读取 |
| **剩余处理** | 循环外单独处理剩的一行或一列：<br> `top === bottom` 剩一行<br> `left === right` 剩一列 | 无需单独处理，剩余部分自然包含在循环体中，<br> 通过条件判断逐步结束 |
| **可读性/维护性** | 与螺旋矩阵 II 生成逻辑高度统一，<br> 熟悉左闭右开者极易理解 | 代码更简洁，变量更新集中，<br> 是 LeetCode 上最常见的写法 |

## 二、它们与螺旋矩阵 II 生成的关系

### 1. spiralOrder1 与螺旋矩阵 II 如出一辙

你在螺旋矩阵 II 中用的典型逻辑是：

```javascript
while (loop--) {
    row = startX; col = startY;
    for (; col < n - offset; col++) { ... }      // 上边：左闭右开
    for (; row < n - offset; row++) { ... }      // 右边：上闭下开
    for (; col > startY; col--) { ... }          // 下边：右闭左开
    for (; row > startX; row--) { ... }          // 左边：下闭上开
    startX++; startY++; offset++;
}
if (n % 2 === 1) { arr[mid][mid] = count; }      // 处理中心点
```

和 `spiralOrder1` 对比：
- **循环条件**：生成用 `loop` 计数（圈数），遍历用 `top < bottom && left < right`（至少能转一圈），本质相同。
- **遍历区间**：生成用 `< n - offset` 和 `> startY` 等实现左闭右开，遍历用 `col < right`、`row < bottom` 等，完全对应。
- **角落处理**：生成时每个角落只被一条边处理，遍历同样如此。
- **剩余部分**：生成最后单独补中间的点（或单行/单列），遍历也单独处理单行/单列。

**结论**：`spiralOrder1` 就是螺旋矩阵 II 的“读取版”。如果你彻底掌握了 II 的生成逻辑，拿过来把“赋值”改成“读取”，其他几乎不用动。

### 2. spiralOrder 是左闭右开的“变体”

虽然 `spiralOrder` 用闭区间读完整行/列，但通过 **读完一条边立刻收缩边界**，巧妙地在**逻辑上等价于左闭右开**：

- 读上边（包含右上角）→ `top++`，右上角不再属于新的右边。
- 读右边（包含右下角）→ `right--`，右下角不再属于新的下边。
- 读下边前判断 `if (top <= bottom)`，若只剩一行就不会回头再读。
- 读左边前判断 `if (left <= right)`，若只剩一列就不会重复。

它依然保证每个元素恰好读取一次，只是**把防止重叠的机制从“显式不读最后一个”变成了“读完立即删除该边界”**。
它和生成的对应没那么直接，但本质思想（一圈一圈、边界向内缩）完全相同。

## 三、核心思想总结（助你完全内化）

### 三种实现其实都在遵循同一个核心模式：

1. **圈层思维**
   把矩阵看成一层层外壳，从外向内逐层处理。

2. **四条边的固定处理顺序**
   无论生成还是遍历，都是 **上 → 右 → 下 → 左** 的顺时针顺序。

3. **边界控制与收缩**
   生成中用 `startX/startY/offset`，遍历中用 `top/bottom/left/right`。每完成一圈，边界向内收缩一格。

4. **角落元素只处理一次**
   - **左闭右开**：通过“每条边少走一个格子”显式实现。
   - **闭区间+边界收缩**：通过“读完一条边后将该边界移走”隐式实现。
   效果完全一致：每个角落恰好属于一条边。

5. **剩余部分的处理**
   - 如果剩一行或一列（或一个点），不能再用“四条边”的方式，需要单独填充或读取。
   - 左闭右开在循环外显式处理；闭区间法通过条件判断使其自然涵盖。

### 一句话总结

> **螺旋矩阵生成和遍历是一体两面**：
> - 生成 = 按“左闭右开”向空矩阵填数
> - 遍历 = 从已有矩阵按相同顺序读数
>
> 无论哪种写法，本质都是维护四条边界，一圈一圈处理，保证每个位置恰好访问一次。
> 你可以把 `spiralOrder1` 当成“生成代码的直接镜像”，把 `spiralOrder` 当成它的简洁等价实现。理解了这一点，螺旋矩阵系列就彻底通了。
